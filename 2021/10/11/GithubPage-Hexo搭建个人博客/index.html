GithubPage+Hexo搭建个人博客

发表于2021-10-11|更新于2021-10-11|Github教程
|字数总计:1.3k|阅读时长:4分钟|阅读量:

前言

个人GitHub仓库在开启GitHub Page属性后，用户可以得到一个域名为”用户名.http://github.io“的个人站点，而这个仓库会作为站点的免费服务器，适合拿来作为个人博客网站。
目前有两种比较主流的方式：
1：GithubPage+Hexo博客框架
模板多，可修改性大，所以我比较喜欢。
2：GithubPage+Jekyll
github官方支持，部署方便，版式比较简单。
所以下文为GithubPage+Hexo的搭建教程。

一、准备环境

1.1安装Node.js

下载地址 https://nodejs.org/zh-cn/downlo

1.2安装Git

参考我的另一教程：Github使用手册
可以通过ssh -T -v git@github.com 访问github即可。
​

安装完毕，查看版本以确认安装成功：

二、部署本地仓库

新建一个本地文件夹作为仓库“blog”其内右键“Git Bash”打开终端
默认软件源网速可能比较慢，所以我们更换一个：
npm config set registry "https://registry.npm.taobao.org"

  • 安装hexo：

npm install hexo-cli -g

  • 初始化hexo：

hexo init

  • 安装其他插件：

npm install

  • 生成hexo默认的静态页面

hexo g

  • 启动本地服务器

hexo s
执行完毕就创建了一个本地博客，在 http://localhost:4000/ 可以查看。

三、基于SSH部署至Github

3.1生成自己的github page

在github中建立一个仓库，仓库的名称【必须】为：
你的帐号名.github.io
github会自动为你创建一个个人页面，进入仓库，在Setting→Page可以直接设置jkeyll主题，就能看到你的github个人页面了。但我们在这里可以不设置。

3.3deploy至github

  • 安装deploy-git命令包

npm install hexo-deployer-git
不安装就没法用部署命令

  • 清除缓存文件

hexo clean
清空上次部署时生成的文件，第一次不用输，以后每次部署前输入，避免问题。

  • 生成静态页面（hexo generate）

hexo g

  • 部署（hexo deploy）

hexo d
【注】最后显示”INFO Deploy done: git”就是完成部署，检查一下执行日志有无异常。第一次可能会遇到”warning: LF will be replaced by CRLF in ……”，这个问题，是因为windows中的换行符为 CRLF ， 而在linux下的换行符为 LF，所以出现符号转义问题，输入以下命令将自动转换关闭就行：
git config --global core.autoCRLF false

四、自定义域名（如果有）

进入仓库，点击Settings→Custom domain，输入你的域名。然后在本地博客目录”source”文件夹中创建一个名为CNAME的无后缀文件，填入自己域名即可。
【注】192.30.252.153 和 192.30.252.154 是GitHub的服务器地址（未证）

五、网站设置（Hexo设置）

通过编辑本地博客目录中的”_config.yml”文件，我们可以设置网站的一些参数。
参考Hexo的官方指南： 文档 | Hexo

六、发布文章

6.1新建文章

在blog文件夹打开git命令行
输入hexo new <模板> <文章名>新建文章
其中<模板>在scaffolds中，默认情况下有三个模板post、draft、page

参数 功能 路径
post 新建文章 /source/_posts/
draft 新建草稿 /source/_drafts/
page 新建页面（标签页，分类页等） /source/

不选择参数默认是 post
创建完毕，在source/_posts/文件夹中多了一个title.md文件。打开可以进行文章的编辑。

6.2文章设置

文章内容中两条—之间的内容为文章的配置内容，可以在其中设置标签、分类等。
通过对文件头部配置信息中tags 与categories项的修改可以设置文章的标签及分类。

其中标签可以按格式设置多个：

6.3上传图片

  • 安装插件

npm install hexo-asset-image --save

  • 打开博客配置文件_config.yml，将post_asset_folder这个选项设置为true

post_asset_floder: true
这样在创建文章后，在/source/_posts文件夹下除了会生成xxx.md文件，还会生成一个同名文件夹存放图片。
