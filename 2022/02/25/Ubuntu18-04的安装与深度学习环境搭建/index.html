换源（加快下载速度

由于Ubuntu 默认使用的官方源的服务器在欧洲，从国内访问速度很慢。出于某些原因，梯子也被封了很多。因此换成国内的源可以极大提高访问速度。国内的阿里、网易以及一些重点高校也都有 Ubuntu 的源，所以在装完 Ubuntu 系统后最好把官方源更换为国内的源。

8.安装pytorch

去官网下载对应cuda版本的pytorch

https://pytorch.org/get-started/previous-versions/

一个坑：

在使用清华源安装pytorch时，总是网络中断。

解决：官方提供的下载命令中

conda install pytorch==1.6.0 torchvision==0.7.0 cudatoolkit=10.0 -c pytorch

其中，-c pytorch参数指定了conda获取pytorch的channel,在此指定为conda自带的pytorch仓库。因此，只需要将-c pytorch语句去掉，就可以使用清华镜像源快速安装pytorch了。

【有时候清华源挂了，就用指定的仓库下吧，更好】

1、ubuntu内核与显卡驱动的适配

ubuntu经常会自动更新内核，这时候的显卡驱动不会更新，所以经常会出现因为二者不适配而出现nvidia-smi命令失效的情况！

这时候！

请趁着还可以操作电脑去更新驱动！！千万不要重启！！不然你就再也进不去界面了QWQ！

如果实在手贱重启了，操作如下：

PS：内核进入操作：

从recover mode下进入，第一项选择resume进入。

旧内核可以进入的情况：

1、从grub界面（也就是ubuntu高级选项界面）选择可以进入的以前的旧内核

2、立马去更新驱动，更新过程可能会出现假死机状态，耐心等待，大概估计下啥时候更新完

3、按魔法键重启，不要强制启动

旧内核也都进不去的情况：

又一次碰到这个问题，但旧内核也进不去了，差点以为完蛋了，还好电脑装了远程，趁卡死前立马启动了远程，通过笔记本是可以正常控制台式机的。

输入nvidia-smi显示

输入 nvcc -V有正常输出，说明驱动在。

果然是显卡驱动出问题了。

这时有两种方法：

1、修复驱动

2、更换旧内核（但我们所有内核都出故障了）

开始修复驱动：

魔法键：

  1. 伸出你的左手，同时按住+键，别松开

  2. 右手先按一下，就是Prt Sc截图键，左手别松开，等1秒

  3. 右手按一下 R，左手别松开，等1秒（记忆指南：busier的倒叙）

  4. 右手按一下 E，左手别松开。这时包括桌面在内，所有程序都会终止，你会看到一个黑乎乎的屏幕，稍微等一段时间

  5. 右手依次按下 I，S，U，B，左手别松开。每按一次都等那么几秒种，你会发现每按一次，屏幕上信息都会有所变化。最后按下B时，屏幕显示reset，这时你的左手可以松开了，等几秒钟，计算机就会安全重启。

2、ubuntu系统突然变成只读文件系统

这个问题也是我突然碰到了，晚上跑训练白天来就出现了这个问题，可能是因为windows+ubuntu双系统，然后导致某个磁盘突然出问题了。解决方式如下：

方法1：

我是先重启了，然后发现ubuntu就进不去了，慌得一笔，然后从windows系统里，

开始菜单-设置-恢复-重启高级启动菜单-立即重启-通过设备-选择你的Ubuntu-等待重启

出现报错：The root filesystem on /dev/sda5 requires a manual fsck

手动修复

(initramfs) fsck /dev/sda1 // 报错提示那块有问题就是sda几

然后一路y

最后切记exit

系统会恢复重启

方法2(我没试过）：

趁着还在ubuntu系统里时，不重启

mount 命令可以看到哪个磁盘变成ro

使用fsck手动修复，具体操作如下：

切换到root用户，运行fsck.ext4 -y /dev/vdb1

说明：文件系统采用fsck.ext4，/dev/vdb1是系统/根分区。运行完毕后reboot重启系统就恢复正常。
